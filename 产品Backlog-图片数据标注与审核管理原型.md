# 产品Backlog（需求清单）

## 图片数据标注与审核管理原型——数据管理方向项目需求分析

> **适用信息**：适合人数 3-4人；难度 中-高；方向 数据管理。
> **项目目标**：面向课程或科研数据，完成图片上传、标签标注、审核、导出和统计。

---

## 〇、角色定义

在阅读本文档前，需明确以下角色的职责与权限边界：

| 角色 | 职责概述 | 核心权限 |
|------|---------|----------|
| **系统管理员** | 全局配置、用户管理、项目管理 | 全量数据可见，创建/归档项目，分配角色，查看全平台统计 |
| **数据管理员** | 数据导入/导出、质量统计、标签体系维护、标注任务分配、自动标注模型配置 | 图片上传/删除，标签体系管理（CRUD），数据导出，查看全量数据，配置自动标注模型（YOLOv8/SAM） |
| **审核员** | 审核标注结果，把控数据质量 | 查看待审核/通过/退回修改数据，执行审核操作，查看审核日志，领取/被指派审核任务 |
| **标注员** | 执行标注任务，修正自动标注结果 | 仅可见已分配的图片（自己所有的数据 + 他人审核通过的数据），进行标注，提交审核，查看自身标注统计 |

> 说明：文档中统一使用以上四种角色。`数据管理员`承担了原部分文档中"标注管理员"、"项目经理"、"任务分配管理员"的职责，不再单独设立冗余角色。

---

## 一、项目概述

本项目旨在构建一个图片数据标注与审核管理原型系统，聚焦于图片数据从上传、标注、审核到导出应用的全生命周期管理。通过整合自动化标注技术与规范化审核流程，实现图片数据的高效流转、质量管控与价值挖掘。

系统面向课程或科研场景，核心功能涵盖：图片数据集创建与上传、标签体系维护与图片标注、标注审核与退回修改、版本记录、按标签检索/统计/导出。可选扩展包括：矩形框/区域标注原型、多人一致性统计、数据脱敏提示、标注进度看板。

## 四、非功能性需求

| ID | 需求描述 | 类型 | 详细说明 |
|----|---------|------|---------|
| **NFR-001** | 并发控制策略 | 性能/可靠性 | 审核操作多为管理员手动执行，不涉及复杂奖励机制或高频并发请求。数据管理层面采用乐观锁机制即可，无需过度设计。审核任务领取机制（PBI-021）提供基本的加锁/解锁策略。 |
| **NFR-002** | 原型简化法实施 | 架构/开发策略 | 系统采用简化原型法思路，优先保障核心数据流转（上传→标注→审核→导出）的功能与界面实现，降低初期开发成本与需求风险。Must Have项对应MVP。 |
| **NFR-003** | 基于角色的访问控制（RBAC） | 安全 | 系统需实现角色-权限模型。标注员仅可查看和修改已分配/自己提交的数据，不可获取其他标注员的未审核数据；标注员修改已审核通过的数据时触发状态重置+版本保留。权限变更实时生效。 |
| **NFR-004** | 标注画布性能 | 性能 | 支持加载至少2000×2000像素的图片，平移/缩放帧率不低于30fps。性能指标按标注复杂度区分：BBox超过500个时，操作响应延迟<200ms；多边形节点总数超过5000个时，操作响应延迟<200ms。 |
| **NFR-005** | 导出大数据量处理 | 可靠性 | 当导出数据总量超过2GB时，系统应提供分卷压缩或异步生成下载链接（支持断点续传），避免HTTP请求超时导致导出失败。 |
| **NFR-006** | 浏览器兼容性 | 可用性 | 保证在最新版Chrome、Edge、Firefox上所有标注工具（多边形绘制、分割画笔、圆形等）行为一致，无功能缺失或样式错乱。 |
| **NFR-007** | 操作日志审计 | 安全 | 所有删除、修改审核状态、导出数据的操作均应记录操作日志（至少保留6个月），日志包含操作人、时间、IP地址、操作类型、操作内容摘要。 |

## 六、术语释义

| 术语 | 释义 |
|------|------|
| YOLO格式 | You Only Look Once，以归一化中心点坐标+宽高表示的标注格式 |
| COCO格式 | Common Objects in Context，以JSON结构化表示的标注格式 |
| VOC格式 | Pascal Visual Object Classes，以XML文件表示的标注格式 |
| SAM模型 | Segment Anything Model，Meta开源的通用图像分割模型 |
| 语义分割 | 将图像中每个像素分配类别标签的像素级分类任务 |
| 边界框（BBox） | 以矩形框标注目标位置的标注方式 |
| IoU | Intersection over Union，交并比，评估两个标注区域重叠程度的指标 |
| RBAC | Role-Based Access Control，基于角色的访问控制 |
| 软删除 | 标记为删除状态但保留数据记录，支持恢复；区别于物理删除（永久清除） |
| 人天 | 1人天 = 8小时，指1名全栈（或前端+后端协作）开发者的日工作量 |
