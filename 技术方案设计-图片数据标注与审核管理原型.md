### 1.2 详细选型与理由

| 层次 | 技术 | 版本 | 选型理由 |
|------|------|------|---------|
| **前端框架** | React | 18.x | 生态成熟，Canvas 标注社区方案丰富（react-konva / fabric.js） |
| **前端语言** | TypeScript | 5.x | 标注数据结构复杂（多态标注类型），类型系统保障重构安全 |

| **ML 模型** | Ultralytics YOLOv8 | latest | pip 包直接调用，CPU 可运行，输出标准格式 |

| **鉴权** | python-jose (JWT) | latest | 无状态 Token，适合前后端分离；RBAC 通过 claims 携带角色 |
| **Web 服务器** | Nginx | latest | 静态资源（React build）+ 反向代理 API + `client_max_body_size` 控制上传 |
| **容器化** | Docker Compose | — | 统一编排前后端+DB+MinIO+Redis+Celery，降低 3-4 人团队环境搭建成本；每容器配置 healthcheck |

### 1.3 关键技术决策

| # | 决策点 | 方案 | 理由 |
|---|--------|------|------|
| 1 | 标注数据结构存储 | JSONB 字段存坐标 + 元数据表存索引字段 | 兼容多态标注（BBox/Polygon/Keypoint/Mask），避免多表 JOIN 爆炸 |
| 2 | 图片存储路径 | MinIO 存储原图 + 缩略图（长边≤300px） | 列表页加载缩略图，标注画布加载原图；S3 兼容便于未来迁移云存储 |
| 3 | 缩略图生成 | Celery 异步任务，`thumbnail_key` 初始为 NULL，生成后回写 | 避免大图上传时同步生成缩略图阻塞接口；生成失败自动重试 3 次，失败后前端用原图 CSS 缩放降级 |
| 4 | 自动标注调用 | Celery 异步任务 + 30s 超时 → 降级手动 | Web 请求不阻塞；超时后前端轮询任务状态并提示 |
| 5 | 自动→人工过渡策略 | 人工编辑自动标注时，原自动标注保留（`is_auto=true`），创建新人工标注记录，原记录 `superseded_by` 指向新记录 | 保留自动标注溯源能力；`annotation_versions` 记录每次变更（含自动标注首次生成） |
| 6 | YOLOv8/SAM 部署 | 开发阶段嵌入 FastAPI；生产独立容器，Celery Worker 通过 HTTP 调用 `/predict` 接口 | 便于独立扩缩容；ML Service 不可用时降级手动；支持 GPU 节点与自动扩缩容 |
| 7 | 并发审核控制 | Redis `SETNX` 原子锁，锁超时 30 秒自动释放；任务领取时在 `task_assignments` 加锁，审核操作校验锁持有者；不采用数据库行锁（长事务易导致连接占用/死锁） | 防止两个审核员同时审核同一标注 |
| 8 | 导出分卷策略 | 单 ZIP > 2GB 或图片数 > 5000 时，自动拆分多个分卷（part1, part2...） | 避免单文件过大导致下载超时；`export_records.volumes` (JSONB) 存储分卷列表 |
| 9 | ML 服务容错 | Celery 任务自动重试 3 次（指数退避 5s/15s/45s）+ 重试耗尽进入 DLQ（Redis 死信队列）；ML Service 连续失败 5 次触发熔断，5 分钟内不再调用 | 避免异步任务静默失败；熔断期间降级为手动标注 |
| 10 | 导出文件生命周期 | Celery Beat 定时任务，每日清理超过 7 天的导出 ZIP 及 `export_records` 记录 | 防止 MinIO 存储无限膨胀 |

### 2.3 模块职责

| 模块 | 职责 | 输入 | 输出 |
|------|------|------|------|
| **Auth Module** | 登录/注册/JWT签发/RBAC拦截 | 凭证 | JWT Token |
| **Project Module** | 项目CRUD、归档、成员管理 | 项目信息 | 项目实体 |
| **Image Module** | 上传/校验/元数据/软删除，异步生成缩略图（Celery），`thumbnail_key` 初始 NULL | 图片文件 | 图片记录 + 存储 URL |
| **Annotation Module** | 标注CRUD（含 task_assignments 越权校验）、版本快照、类别绑定、自动标注触发，自动→人工过渡逻辑 | 标注几何 + 类别ID | 标注记录 + 版本记录 |
| **Category Module** | 标签体系CRUD、层级管理、导入导出 | 类别定义 | 类别实体 |
| **Review Module** | 审核状态机、任务领取/指派（任务领取时 Redis 加锁，审核操作校验锁持有者）、审核日志 | 审核动作 | 审核记录 + 日志 |
| **Cleanup Module** | Celery Beat 定时清理：导出文件（>7天）删除 + `export_records` 归档 | 定时触发 | 清理日志 |
| **Export Module** | 格式转换 (YOLO/COCO/VOC)、ZIP打包、>2GB或>5000张自动分卷 | 项目+筛选条件 | ZIP 下载链接 + volumes 分卷列表 |
| **Stats Module** | 标注进度、准确率、用户画像、一致性 | 统计参数 | 聚合数据 |
| **ML Module** | YOLOv8 推理、SAM 推理、超时管理；生产环境独立容器 HTTP 调用 | 图片 + 模型 | 标注结果 |

### 2.4 核心数据流

```
上传流程:
  用户 → Nginx → FastAPI → 文件校验 → MinIO(存原图) → PostgreSQL(元数据, thumbnail_key=NULL)
       → 异步 Celery 任务生成缩略图 → MinIO(缩略图) → 更新 PostgreSQL(thumbnail_key)
       → 生成失败则重试 3 次 → 仍失败则前端使用原图 CSS 缩放降级

标注流程:
  标注员 → 打开图片(Fabric.js画布) → AnnotationService 校验 task_assignments（已分配且未过期）
       → 绘制几何 → 绑定类别 → 保存 → PostgreSQL(标注JSONB+版本)
       → 提交审核 → 状态='pending'
  若标注的是自动标注结果（is_auto=true）：人工编辑时保留原记录，创建新人工记录，
  原记录 superseded_by 指向新记录ID

审核流程:
  审核员 → 领取/指派任务 → task_assignments 加 Redis 锁（SETNX, 超时30s）
       → 查看标注(左右对照) → 通过/退回
       → PostgreSQL(更新状态) → 释放锁 → 审核日志 → 通知标注员

导出流程:
  数据管理员 → 选择项目/筛选 → 选择格式(YOLO/COCO/VOC)
       → FastAPI → 异步 Celery 任务 → 遍历标注数据 → 格式转换
       → 预估总量: 若>2GB或>5000张则拆分为多个ZIP分卷
       → 打包 → MinIO → 更新 export_records.volumes(分卷列表) → 返回下载链接

自动标注流程(YOLOv8 + SAM):
  数据管理员 → 触发自动标注 → Celery 任务入队
       → [开发环境] 直接调用 YOLOv8/SAM 推理
       → [生产环境] HTTP 请求 ML Service /predict 接口
       → 失败自动重试 3 次（指数退避 5s/15s/45s）→ 仍失败则进入 DLQ
       → ML Service 连续 5 次失败触发熔断（5 分钟内降级手动标注）
       → 结果转标注格式(is_auto=true) → PostgreSQL → 前端轮询完成

交互式分割流程(SAM 实时):
  标注员 → 在画布上点击正/负样本点 → WebSocket 发送坐标
       → ML Service 实时推理 → WebSocket 返回多边形轮廓
       → 前端渲染交互式分割结果 → 用户确认/调整 → 保存为 segmentation 标注

文件清理流程(Celery Beat):
  每日 02:00 → 扫描 export_records(completed, created_at < now()-7d)
       → MinIO 删除 ZIP 文件 → 标记 export_records.status='expired'
       → 保留 audit_log 中的导出记录
```

### 3.2 建表 SQL（核心表）

```sql
-- ============================================================
-- 1. 用户表
-- ============================================================
CREATE TABLE users (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username      VARCHAR(64)  NOT NULL UNIQUE,
    password_hash VARCHAR(256) NOT NULL,
    role          VARCHAR(32)  NOT NULL DEFAULT 'annotator'
                  CHECK (role IN ('admin','data_manager','reviewer','annotator')),
    display_name  VARCHAR(128),
    email         VARCHAR(256),
    avatar_url    VARCHAR(512),
    is_active     BOOLEAN      NOT NULL DEFAULT TRUE,
    created_at    TIMESTAMPTZ  NOT NULL DEFAULT now(),
    updated_at    TIMESTAMPTZ  NOT NULL DEFAULT now()
);

-- ============================================================
-- 2. 项目表
-- ============================================================
CREATE TABLE projects (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name          VARCHAR(256) NOT NULL,
    description   TEXT,
    status        VARCHAR(32)  NOT NULL DEFAULT 'active'
                  CHECK (status IN ('active','archived')),
    created_by    UUID REFERENCES users(id),
    created_at    TIMESTAMPTZ  NOT NULL DEFAULT now(),
    updated_at    TIMESTAMPTZ  NOT NULL DEFAULT now()
);

-- 项目成员中间表
CREATE TABLE project_members (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id    UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    user_id       UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    project_role  VARCHAR(32) NOT NULL DEFAULT 'annotator'
                  CHECK (project_role IN ('manager','reviewer','annotator')),
    UNIQUE (project_id, user_id)
);

-- ============================================================
-- 3. 图片表（完整枚举 + CHECK 约束）
-- ============================================================
CREATE TABLE images (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id      UUID         NOT NULL REFERENCES projects(id),
    original_name   VARCHAR(512) NOT NULL,
    storage_key     VARCHAR(512) NOT NULL,        -- MinIO object key（原图）
    thumbnail_key   VARCHAR(512),                  -- 缩略图 MinIO key（初始 NULL，Celery 异步生成后回写）
    width           INTEGER,
    height          INTEGER,
    file_size       BIGINT,                        -- 字节数
    mime_type       VARCHAR(64)  DEFAULT 'image/jpeg',

    -- 标注状态（完整枚举）
    annotation_status VARCHAR(32) DEFAULT 'unannotated'
                    CHECK (annotation_status IN ('unannotated','annotating','annotated','under_review')),

    -- 审核状态（与 annotations.review_status 保持一致，应用层同步）
    review_status   VARCHAR(32)  DEFAULT 'pending'
                    CHECK (review_status IN ('pending','pre_approved','approved','rejected','rework')),

    uploaded_by     UUID REFERENCES users(id),
    deleted_at      TIMESTAMPTZ,                   -- 软删除标记
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- ============================================================
-- 4. 类别（标签）表
-- ============================================================
CREATE TABLE categories (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id    UUID         NOT NULL REFERENCES projects(id),
    name          VARCHAR(128) NOT NULL,
    color         VARCHAR(7)   DEFAULT '#3388FF',   -- HEX color
    parent_id     UUID REFERENCES categories(id),   -- 层级结构
    sort_order    INTEGER      DEFAULT 0,
    shortcut_key  VARCHAR(16),                       -- 快捷键绑定
    is_active     BOOLEAN      DEFAULT TRUE,         -- 停用≠删除
    created_at    TIMESTAMPTZ  NOT NULL DEFAULT now(),
    UNIQUE (project_id, name)
);

-- ============================================================
-- 5. 标注表（核心）
-- ============================================================
CREATE TABLE annotations (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    image_id        UUID         NOT NULL REFERENCES images(id),
    category_id     UUID         REFERENCES categories(id),
    annotator_id    UUID         NOT NULL REFERENCES users(id),
    
    -- 标注类型
    type            VARCHAR(32)  NOT NULL
                    CHECK (type IN ('bbox','polygon','point','line','circle','segmentation')),
    
    -- 几何数据 (JSONB — 多态字段)
    geometry        JSONB        NOT NULL,
    /*
      type='bbox':        {"x":100.0,"y":200.0,"width":150.0,"height":120.0}
      type='polygon':     {"points":[[x1,y1],[x2,y2],...]}
      type='point':       {"x":100.0,"y":200.0,"label":"left_eye"}
      type='line':        {"x1":100,"y1":200,"x2":300,"y2":400}
      type='circle':      {"cx":200,"cy":300,"radius":50}
      type='segmentation':{"mask_key":"masks/img_001_seg_01.png","contours":[[x1,y1],...]}
    */
    
    -- 元数据字段
    is_auto         BOOLEAN      DEFAULT FALSE,     -- 是否自动标注生成
    confidence      FLOAT,                           -- 自动标注置信度 (0~1)
    is_latest       BOOLEAN      DEFAULT TRUE,       -- 是否为当前有效标注（被取代时置 FALSE）
    superseded_by   UUID REFERENCES annotations(id), -- 自动标注被人工替换时指向新人工标注
    version         INTEGER      DEFAULT 1,          -- 当前版本号
    review_status   VARCHAR(32)  DEFAULT 'pending'
                    CHECK (review_status IN ('pending','approved','rejected','rework')),
    created_at      TIMESTAMPTZ  NOT NULL DEFAULT now(),
    updated_at      TIMESTAMPTZ  NOT NULL DEFAULT now()
);

-- ============================================================
-- 6. 标注版本快照表
-- ============================================================
CREATE TABLE annotation_versions (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    annotation_id   UUID         NOT NULL REFERENCES annotations(id),
    version         INTEGER      NOT NULL,
    geometry        JSONB        NOT NULL,          -- 该版本的坐标快照
    category_id     UUID REFERENCES categories(id),
    type            VARCHAR(32)  NOT NULL,
    change_summary  VARCHAR(512),                   -- 变更摘要
    source          VARCHAR(32)  DEFAULT 'manual',  -- 'auto' | 'manual'（区分自动标注首次生成与人工编辑）
    created_by      UUID         NOT NULL REFERENCES users(id),
    created_at      TIMESTAMPTZ  NOT NULL DEFAULT now(),
    UNIQUE (annotation_id, version)
);

-- ============================================================
-- 7. 审核记录表（增加并发锁字段）
-- ============================================================
CREATE TABLE review_records (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    annotation_id   UUID         NOT NULL REFERENCES annotations(id),
    reviewer_id     UUID         NOT NULL REFERENCES users(id),
    action          VARCHAR(32)  NOT NULL
                    CHECK (action IN ('approved','rejected','pre_approved','reset','rollback')),
    reason          TEXT,                            -- 退回原因
    preset_reason   VARCHAR(64),                     -- 预定义原因编码: 'bbox_offset'|'wrong_category'|'missing_label'|'incomplete'|'other'
    triggered_by    VARCHAR(64),                     -- 'system_auto' 或 NULL（人工）
    created_at      TIMESTAMPTZ  NOT NULL DEFAULT now()
);

-- 注：审核锁统一由 task_assignments 表管理（任务领取时 Redis SETNX 加锁），
-- review_records 不再冗余存储锁字段。审核操作时校验当前用户 = task_assignments.assignee_id。

-- ============================================================
-- 8. 任务分配表（标注+审核统一管理）
-- ============================================================
CREATE TABLE task_assignments (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    image_id        UUID         NOT NULL REFERENCES images(id),
    assignee_id     UUID         NOT NULL REFERENCES users(id),
    task_type       VARCHAR(32)  NOT NULL
                    CHECK (task_type IN ('annotation','review')),
    status          VARCHAR(32)  DEFAULT 'assigned'
                    CHECK (status IN ('assigned','in_progress','completed','unlocked')),
    assigned_by     UUID REFERENCES users(id),
    locked_until    TIMESTAMPTZ,                    -- Redis 锁过期时间：审核任务 +30s，标注任务 +24h
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    UNIQUE (image_id, assignee_id, task_type)
);

-- ============================================================
-- 9. 导出记录表（增加分卷字段）
-- ============================================================
CREATE TABLE export_records (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id      UUID         NOT NULL REFERENCES projects(id),
    created_by      UUID         NOT NULL REFERENCES users(id),
    export_format   VARCHAR(32)  NOT NULL
                    CHECK (export_format IN ('yolo','coco','voc')),
    filter_criteria JSONB,                          -- 导出时筛选条件
    file_url        VARCHAR(1024),                  -- MinIO 主下载链接
    file_size       BIGINT,                         -- 总大小（所有分卷之和）
    volumes         JSONB,                          -- 分卷列表: [{"part":1,"url":"...","size":500MB},{...}]
    status          VARCHAR(32)  DEFAULT 'processing'
                    CHECK (status IN ('processing','completed','failed')),
    error_msg       TEXT,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    completed_at    TIMESTAMPTZ
);

-- ============================================================
-- 10. 操作审计日志表
-- ============================================================
CREATE TABLE audit_logs (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID REFERENCES users(id),
    action          VARCHAR(64)  NOT NULL,          -- 'delete_image','change_review','export',...
    target_type     VARCHAR(64),
    target_id       UUID,
    detail          JSONB,
    ip_address      VARCHAR(64),
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- ============================================================
-- 索引汇总（高频查询优化）
-- ============================================================
CREATE INDEX idx_users_role ON users(role);

-- images
CREATE INDEX idx_images_project ON images(project_id);
CREATE INDEX idx_images_review_status ON images(project_id, review_status);
CREATE INDEX idx_images_annotation_status ON images(project_id, annotation_status);
CREATE INDEX idx_images_project_status ON images(project_id, deleted_at, annotation_status, review_status);

-- categories
CREATE INDEX idx_categories_project ON categories(project_id);
CREATE INDEX idx_categories_parent ON categories(parent_id);

-- annotations
CREATE INDEX idx_annotations_image ON annotations(image_id);
CREATE INDEX idx_annotations_category ON annotations(category_id);
CREATE INDEX idx_annotations_annotator ON annotations(annotator_id, created_at);
CREATE INDEX idx_annotations_review ON annotations(review_status);
CREATE INDEX idx_annotations_type ON annotations(type);
CREATE INDEX idx_annotations_image_review ON annotations(image_id, review_status);
CREATE INDEX idx_annotations_image_latest ON annotations(image_id, is_latest) WHERE is_latest = TRUE;
CREATE INDEX idx_annotations_geometry_gin ON annotations USING GIN (geometry jsonb_path_ops);

-- review_records
CREATE INDEX idx_review_annotation ON review_records(annotation_id, created_at DESC);

CREATE INDEX idx_audit_user_time ON audit_logs(user_id, created_at DESC);
CREATE INDEX idx_audit_created ON audit_logs(created_at);

-- export_records
CREATE INDEX idx_export_project ON export_records(project_id, created_at DESC);
```
