## 1. 系统概述

本平台是一个基于 Web 的**图片数据标注与团队审核管理系统**，适用于：

- 深度学习训练数据集的标注制作
- 多人协作标注 + 审核质量控制
- YOLOv8 AI 辅助自动标注
- 多格式数据导出（YOLO / COCO / VOC）

**技术能力：** 支持矩形框 (BBox) 和多边形 (Polygon) 两种标注类型，坐标存储为绝对像素值，支持版本历史追溯。

## 2. 安装与启动

### 2.1 环境要求

| 组件 | 要求 |
|------|------|
| Python | 3.11+ |
| Node.js | 20+ |
| 数据库 | SQLite（本地开发）/ PostgreSQL（生产） |
| ML（可选） | CUDA GPU + PyTorch（YOLO 自动标注） |

### 2.2 本地开发启动

**第一步：启动后端**

```bash
cd annotation-platform/backend
pip install -r requirements.txt
cp .env.example .env
# 默认使用 SQLite，无需额外配置数据库
uvicorn app.main:app --reload --port 8000
```

后端运行在 `http://localhost:8000`。

**第二步：启动前端**

```bash
cd annotation-platform/frontend
npm install
npm run dev
```

前端运行在 `http://localhost:5173`，浏览器打开即可使用。

**第三步（可选）：启动 ML 自动标注服务**

```bash
cd annotation-platform/ml_service
pip install -r requirements.txt
uvicorn api:app --port 8001
```

### 2.3 Docker 部署

```bash
cd annotation-platform
cp .env.example .env
# 基础服务
docker compose up -d postgres redis minio backend celery-worker
# 前端
docker compose up -d frontend
# 生产模式（含 Nginx）
docker compose --profile prod up -d
# ML 服务（需 GPU）
docker compose --profile ml up -d ml-service
```

### 2.4 注册管理员账号

1. 打开浏览器访问 `http://localhost:5173`
2. 点击"注册"创建账号（首次注册自动成为标注员）
3. 如需管理员权限，需在数据库中手动修改 `role` 字段为 `admin`

```sql
UPDATE users SET role = 'admin' WHERE username = '你的用户名';
```

## 3. 用户角色与权限

| 角色 | 英文标识 | 可执行操作 |
|------|---------|-----------|
| **管理员** | `admin` | 全部操作：项目管理、用户管理、上传图片、标注、审核、导出、统计、删除 |
| **数据管理员** | `data_manager` | 项目管理、上传图片、自动标注、数据导出、删除图片 |
| **审核员** | `reviewer` | 浏览项目、审核队列、审核通过/退回 |
| **标注员** | `annotator` | 标注图片、提交审核、查看自己上传/分配的图片 |

**可见性规则：** 标注员只能看到自己上传的图片、被分配的图片、以及已通过审核的图片。

## 4. 快速入门

以下是一个完整的标注与审核工作流（约 15 分钟上手）：

```
1. 管理员: 创建项目 → 上传图片 → 创建类别
2. 标注员: 打开项目 → 选择图片 → 标注 → 保存 → 提交审核
3. 审核员: 审核队列 → 领取任务 → 审核通过/退回
4. 管理员: 数据导出 → 下载标注数据集
```

**快捷键速查（标注工作台）：**

| 按键 | 功能 |
|------|------|
| `V` | 选择模式 |
| `B` | 矩形框工具 |
| `P` | 多边形工具 |
| `Delete / Backspace` | 删除选中的标注 |
| `Ctrl + Z` | 撤销 |
| `Ctrl + Y` | 重做 |
| `Ctrl + S` | 保存 |
| `空格 + 拖拽` | 平移画布 |
| `滚轮` | 缩放 |

## 5. 功能详解

### 5.1 仪表盘

登录后的默认页面，显示系统概览信息和导航入口。

### 5.2 项目管理

**路径：** 侧边栏 → 项目管理

- **创建项目：** 点击"新建项目"，填写项目名称和描述
- **进入项目：** 点击项目卡片进入，可看到图片列表、类别管理、导出等功能

### 5.3 图片管理

**路径：** 项目管理 → 选择项目 → 进入图片列表

**上传图片：**

1. 点击右上角"上传图片"按钮
2. 选择 JPG 或 PNG 文件（支持多选）
3. 等待上传完成，自动生成缩略图

**筛选与搜索：**

- **文件名搜索：** 在搜索框输入关键词，按回车搜索
- **标注状态筛选：** 未标注 / 已标注 / 审核中
- **审核状态筛选：** 待审核 / 已通过 / 已退回

**图片操作按钮：**

| 按钮 | 角色 | 功能 |
|------|------|------|
| 点击卡片 | 标注员/管理员 | 进入标注工作台 |
| 点击卡片 | 审核员 | （待审核图片）进入审核面板 |
| 自动标注 | 管理员/数据管理员 | AI 辅助标注 |
| 审核 | 审核员 | 进入审核面板 |
| 🗑️ | 管理员/数据管理员 | 删除（移入回收站） |

### 5.4 类别管理

**路径：** 项目管理 → 选择项目 → 类别管理（侧边栏或项目内Tab）

类别用于对标注对象进行分类（如：裂缝、坑洞、车辆、行人）。

- **创建类别：** 填写名称、选择颜色、设置快捷键
- **类别树：** 支持父子层级，如：路面病害 → 裂缝、坑洞
- **颜色标记：** 每种标注在画布上以对应类别颜色显示

### 5.5 标注工作台

**路径：** 图片列表 → 点击图片卡片

#### 界面布局

```
┌──────────────────────────────────────────────────┐
│ ← 返回   撤销/重做   缩放%   标注数   [保存] [提交审核] │  ← 工具栏
├────┬──────────────────────────────────┬───────────┤
│ 工 │                                  │ 类别列表  │
│ 具 │         画布区域                  │           │
│ 栏 │    （图片 + 标注可视化）          │ 标注列表  │
│    │                                  │           │
├────┴──────────────────────────────────┴───────────┤
```

#### 绘制矩形框 (BBox)

1. 按 `B` 键或点击左侧工具栏"矩形"按钮
2. 在图片上按住鼠标左键**拖拽**出一个矩形
3. 松开鼠标完成绘制
4. 在右侧**类别列表**中选择对应类别（点击类别名称）
5. 标注会自动关联到选中类别并更改颜色

#### 绘制多边形 (Polygon)

1. 按 `P` 键或点击左侧工具栏"多边形"按钮
2. 在图片上**逐一点击**添加顶点（至少3个顶点）
3. **双击**最后一个点或按 `Enter` 键闭合多边形
4. 按 `Esc` 取消绘制
5. 在右侧类别列表中为多边形绑定类别

#### 编辑标注

- **移动：** 切换到选择模式(`V`)，拖拽标注框
- **调整大小：** 拖动标注框的角/边控制点
- **删除：** 选中标注后按 `Delete` 或 `Backspace`
- **修改类别：** 选中标注 → 点击新类别
- **撤销/重做：** `Ctrl+Z` / `Ctrl+Y`

#### 保存与提交

- **保存：** 点击"保存"按钮或 `Ctrl+S`，保存当前标注进度
- **提交审核：** 点击"提交审核"按钮，图片状态变为"待审核"
- 工具栏右侧会显示"未保存"标记，提醒保存

> **重要提示：** 提交审核前请确保所有标注已保存。提交后标注将进入审核队列，无法再编辑。

### 5.6 自动标注 (YOLOv8)

**前提条件：** 需要 ML 微服务运行中。

**使用步骤：**

1. 在图片列表中，点击图片卡片上的"自动标注"按钮
2. 系统调用 YOLOv8 模型进行推理
3. 自动生成的标注以**橙色虚线框**显示
4. 自动标注结果可以手动调整（拖拽/缩放后自动转换为手动标注）

**模型说明：**

| 模型 | 特点 |
|------|------|
| `yolov8n` | 轻量快速，适合通用目标检测 |
| `yolov8s` | 精度更高，速度略慢 |

> 自动标注结果仅作为辅助参考，建议人工检查并修正后提交审核。

### 5.7 审核系统

**路径：** 图片列表 → 待审核图片自动显示"审核"按钮

#### 审核流程

```
标注员提交 → 审核队列(pending) → 审核员领取(claim)
                                     │
                           ┌─────────┴─────────┐
                           ↓                   ↓
                        通过(approved)      退回(rejected)
                           ↓                   ↓
                       完成归档           标注员修改后重新提交
```

#### 审核面板操作

1. 在图片列表中找到待审核图片（标记"待审核"），点击"审核"按钮
2. 审核面板显示**双画布对比**：
   - 左侧：原始图片
   - 右侧：标注结果（标注框叠加显示）
3. 右侧显示标注列表，包含：类别、类型（自动/手动）、置信度
4. 在"审核意见"框中填写备注（通过时选填，退回时**必填**）
5. 点击"通过"或"退回"完成审核

#### 任务锁定机制

- 审核员点击审核时，系统自动锁定该任务 **30 分钟**
- 锁定期间其他审核员无法操作同一任务
- 离开审核页面时自动释放锁定

### 5.8 数据导出

**路径：** 项目管理 → 选择项目 → 导出

#### 支持的导出格式

| 格式 | 输出内容 | 适用场景 |
|------|---------|---------|
| **YOLO** | `images/` + `labels/` + `classes.txt` | YOLOv5/v8/v11 训练 |
| **COCO** | `annotations.json`（images + annotations + categories） | Detectron2、MMDetection |
| **VOC** | `JPEGImages/` + `Annotations/`（PASCAL VOC XML） | 经典目标检测框架 |

#### 导出步骤

1. 选择导出格式（YOLO / COCO / VOC）
2. 可选设置过滤条件（只导出已标注的、只导出已通过审核的）
3. 点击"开始导出"
4. 导出任务在后台异步执行，状态自动更新
5. 完成后点击"下载"按钮获取 ZIP 文件

#### 坐标说明

- **YOLO 格式：** 坐标自动归一化到 [0, 1]（cx, cy, w, h 均为相对值）
- **COCO 格式：** 坐标保留为绝对像素值
- **VOC 格式：** 坐标保留为绝对像素值（整数）

### 5.9 统计仪表盘

**路径：** 侧边栏 → 人员统计 → 选择项目

#### 数据卡片

| 卡片 | 内容 |
|------|------|
| 图片总数 | 已标注数、待标注数 |
| 标注总数 | 自动/手动标注数、平均每图标注数 |
| 审核通过率 | 通过数、退回数、待审核数 |
| 标注进度 | 进度条 + 百分比 |

#### 图表

| 图表 | 类型 | 说明 |
|------|------|------|
| 30天趋势图 | 折线图 | 每日新增标注量和审核量 |
| 类别分布 | 饼图 | 各类别标注数量的占比 |
| 标注员表现 | 堆叠柱状图 + 表格 | 每个标注员的通过/退回/自动/手动标注数及准确率 |

### 5.10 回收站

**路径：** 侧边栏 → 回收站

- 选择项目查看已删除的图片
- **恢复：** 点击"恢复"按钮，图片回到图片列表
- **彻底删除：** 点击"彻底删除"按钮（**不可撤销**，同时删除存储文件和关联数据）

## 6. 标注规范

### 6.1 标注原则

1. **紧贴目标：** 边界框应紧贴目标物体外轮廓，不包含过多背景
2. **完整性：** 确保目标物体完全在边界框内
3. **一致性：** 同类目标的标注方式保持一致（使用相同的类别）
4. **不漏标：** 图片中所有可见的目标物体都应标注
5. **不重标：** 同一目标不应有重复标注

### 6.2 多边形使用场景

多边形标注适用于**不规则形状**的目标物体：

- 路面裂缝（非矩形形状）
- 复杂轮廓的物体
- 需要精确分割的场景

### 6.3 审核标准

| 审核结果 | 适用条件 |
|---------|---------|
| **通过** | 标注准确、类别正确、无漏标 |
| **退回** | 标注位置不准、类别错误、漏标、边界框过大过小 |

退回时必须填写具体原因，方便标注员修改。

## 7. 常见问题

### 7.1 如何批量上传图片？

在图片列表页点击"上传图片"，支持多选文件一次性上传。

### 7.2 标注后看不到标注框？

检查右侧类别列表是否已选择类别，标注框颜色可能与背景相近。自动标注的结果为橙色虚线框。

### 7.3 如何修改已提交审核的标注？

如果标注已被退回（状态为"已退回"），图片会自动回到"未标注"状态，标注员可以重新进入标注工作台修改后再次提交。

### 7.4 导出的 YOLO 标签中 class_id 是如何确定的？

class_id 根据项目中的类别排序自动分配（按类别名称字母序），对应关系保存在导出的 `classes.txt` 文件中。

### 7.5 支持哪些图片格式？

支持 `image/jpeg` 和 `image/png` 格式。上传时自动校验文件头，防止伪造格式。

### 7.6 数据库在哪里？

- **本地开发：** SQLite 数据库文件 `annotation-platform/backend/dev.db`
- **Docker 部署：** PostgreSQL 容器中的数据卷

### 7.7 如何备份数据？

1. 备份数据库文件（SQLite 直接复制 `.db` 文件）
2. 备份 `uploads/` 目录（存储的图片、缩略图、导出文件）
3. 建议定期通过数据导出功能保存标注数据
